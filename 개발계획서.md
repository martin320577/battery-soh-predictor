# 폐배터리 SOH 진단 AI — 개발 계획서

## 1. 프로젝트 개요

- 프로젝트명: 폐배터리 건강 상태(SOH) 빠른 진단 AI
- 목표: 짧은 충방전 측정 데이터(수 분)만으로 배터리 SOH를 정확히 예측하는 AI 모델 개발
- 최종 산출물: 웹 기반 진단 대시보드

---

## 2. 배경 및 필요성

전기차 보급 확대로 폐배터리가 급증하고 있다.
폐배터리를 재사용(ESS 등)할지, 소재를 회수(재활용)할지 판단하려면 배터리의 건강 상태(SOH)를 정확히 알아야 한다.

기존 방식은 완전 충방전을 수행해야 하므로 수 시간이 소요된다.
본 프로젝트는 AI를 활용하여 짧은 측정만으로 SOH를 추정함으로써 진단 시간을 대폭 단축하는 것을 목표로 한다.

---

## 3. 사용 데이터

- 데이터셋: NASA PCoE (Prognostics Center of Excellence) 리튬이온 배터리 데이터셋
- 내용: 여러 리튬이온 셀을 수명 끝까지 반복 충방전한 실험 데이터
- 포함 정보: 전압, 전류, 온도, 측정 시간, 용량
- 출처: NASA 공식 데이터 저장소

## 5. 개발 단계

### Phase 0. 환경 세팅 (Day 1)

1. Python 설치
2. VS Code 설치 (코드 편집기)
3. 가상환경 생성
4. 필요한 라이브러리 설치
5. 프로젝트 폴더 구조 생성

### Phase 1. 데이터 준비 (Day 2-3)

1. NASA 데이터셋 다운로드
2. 데이터 구조 확인 (Jupyter 노트북)
   - 파일 열어서 컬럼 확인
   - 데이터 개수, 값 범위 파악
3. 충방전 곡선 시각화
   - x축: 시간, y축: 전압
   - 사이클 경과에 따른 용량 감소 확인
4. 데이터 정제 (결측치, 이상치 제거)

목표: 데이터의 구조와 특성을 이해한다.

### Phase 2. 특성 추출 (Day 4-5)

각 충방전 사이클에서 의미 있는 특성을 추출한다:
- 특정 전압 구간(3.5V → 4.0V) 도달 시간
- 전압 곡선 기울기
- 최고 온도
- 내부저항 추정값 (V/I)

추출된 특성을 표(DataFrame)로 정리한다:

| 사이클 | 도달시간 | 기울기 | 온도 | SOH(%) |
|--------|---------|--------|------|--------|
| 1 | 120s | 0.8 | 25.3 | 100% |
| 50 | 135s | 0.7 | 27.1 | 92% |
| 100 | 160s | 0.5 | 30.2 | 81% |

목표: AI 모델에 입력할 수 있는 정제된 데이터 테이블을 완성한다.

### Phase 3. 모델 학습 (Day 6-8)

1. 데이터 분할: 학습용 80%, 테스트용 20%
2. 모델 학습 및 비교
   - 1차: Random Forest
   - 2차: XGBoost
   - 3차 (선택): LSTM 신경망
3. 성능 평가
   - 실제 SOH vs 예측 SOH 비교 그래프
   - 평가 지표: MAE, RMSE
   - 목표 정확도: SOH 오차 5% 이내
4. 하이퍼파라미터 튜닝을 통한 성능 개선

목표: 데이터를 입력하면 SOH를 출력하는 예측 모델을 완성한다.

### Phase 4. 대시보드 개발 (Day 9-11)

1. Streamlit 기본 문법 학습
2. 화면 구성
   - 충방전 데이터 파일 업로드 기능
   - SOH 예측 결과 수치 표시
   - 배터리 등급 표시 (A: 재사용 / B: 제한적 재사용 / C: 재활용 / D: 폐기)
   - 용량 감소 그래프 및 주요 영향 요인 시각화
3. 학습된 모델 연결 (업로드 → 특성 추출 → 예측 → 결과 표시)

목표: 브라우저에서 작동하는 웹 애플리케이션을 완성한다.

### Phase 5. 마무리 (Day 12-14)

1. 코드 정리 및 주석 작성
2. README.md 작성 (프로젝트 설명서)
3. GitHub 업로드
4. (선택) Streamlit Cloud 배포 → 외부 공유 가능한 링크 생성
